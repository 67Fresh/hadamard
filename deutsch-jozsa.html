DEUTSCH-JOZSA

Given a black-box function f(x), is it constant (same output for all inputs) or balanced (0 for half, 1 for half)? Classical computers need up to 2ⁿ⁻¹+1 queries. The quantum algorithm needs exactly one.

Choose an oracle function f(x)
What the algorithm does
Select an oracle above. The algorithm puts all input qubits into superposition, applies the oracle (which encodes f into quantum phase via phase kickback), then applies Hadamard gates again. Interference causes all amplitudes to cancel to zero if the function is constant, or to partially survive if it's balanced. Measuring the input register gives all zeros for constant, and at least one non-zero bit for balanced — in a single shot.

Understanding Deutsch-Jozsa
Why does a classical computer need so many queries?
In the worst case, you query f(0), f(1), f(2)... and keep getting the same result. You can't conclude "constant" until you've seen more than half the inputs all agree — because if even one output differs, the function is balanced. For n input bits, that's 2ⁿ⁻¹+1 queries in the worst case.
How does interference give the answer in one query?
The Hadamard gate puts all inputs into superposition simultaneously. The oracle applies phase kickback — writing f(x) into the phase of each amplitude. If f is constant, all phases are the same, and the final Hadamard layer causes constructive interference back to |000...0⟩ (all zeros). If f is balanced, half the phases are flipped, causing destructive interference that prevents the all-zeros outcome. Measurement distinguishes the two cases instantly.
Is this a practical speedup?
Not directly — the problem is artificial (no real application requires knowing whether a function is constant or balanced). But it was the first rigorous proof that quantum computers can solve a specific problem exponentially faster than any classical algorithm — establishing the theoretical possibility of quantum advantage. It paved the way for Shor's algorithm, Grover's algorithm, and the whole field of quantum algorithm design.
Who invented it?
David Deutsch proposed the 1-bit version (Deutsch's problem) in 1985 as part of his foundational paper on quantum computation. Richard Jozsa extended it to n bits in 1992. The algorithm is typically named Deutsch-Jozsa after both.
